This rule detects UnutilizedAbstraction between classes. If classes don't use or be used. You will get the issue. It means you should make sure that is it really need to exist.

Noncompliant Example

	class A {
		
	}

Recommended Solution

You have two ways to deal with the smell.

1.Remove the class.
2.Refactor the code,let class has a clearer division of labor. After that,you can get clearer architecture.
