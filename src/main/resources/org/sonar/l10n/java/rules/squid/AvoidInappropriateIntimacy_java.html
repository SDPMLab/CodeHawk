This rule detects Inappropriate Intimacy between classes. If One class uses the internal fields and methods of another class. You will get the issue.

Noncompliant Example

	class A {
		private int a = 1;
		
		public int m1(){
			return a;
		}
		
		void m2(){
			B.m5();
		}
		void m3(){
			B.m5();
		}
		void m4(){
			B.m5();
		}
	}
	
	class B {
		private int b = 1;
		
		public int m5(){
			return b;
		}
		void m6(){
			A.m2();
		}
		void m7(){
			A.m2();
		}
		void m8(){
			A.m2();
		}
	}

Recommended Solution

You have 2 ways to deal with the smell.

1.Replace Delegation with Inheritance.
2.Use Extract Class and Hide Delegate on the class to make the code relations official.
