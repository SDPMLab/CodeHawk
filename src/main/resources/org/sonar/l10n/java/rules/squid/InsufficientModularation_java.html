This rule detects modularation insufficiency within classes.

Recommended Solution

You have three ways to deal with the smell.

1.Migrate functionality to data classes if you encounter a complexity problem. (reference: The Blob solution)
2.Refer the LAVA FLOW antipattern, it can help you to redesign and establish well-designed interfaces so that you won't face this smell again.
3.Refer the Cut-and-Paste Programming antipattern. It is believed that the programmer is lack of OO's practice, hence you may try to refactor to develop a standard version all over again.
