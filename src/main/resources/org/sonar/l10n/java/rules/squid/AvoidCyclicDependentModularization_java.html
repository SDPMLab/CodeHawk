This rule detects class cyclic dependent.
When two or more abstractions have direct or indirect dependencies on each other, they will be detected as cyclic dependency.

Noncompliant Code Example

    class A {
        A(){}
        B bb = new B();
    }

    class B {
        B(){
            A a = new A();
        }
    }

class B will be marked as cyclic dependent.

Recommended Solution

Redesign the components properly so their hierarchy is well designed and there is no need for circular dependencies.
