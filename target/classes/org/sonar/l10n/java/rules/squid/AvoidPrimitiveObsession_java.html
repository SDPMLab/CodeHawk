This rule detects Primitive Obsession code smell in Java.
When too many primitive type (including wrapper classes) parameters were passed, the method will be marked as primitive obsession.
Details of Primitve Obsession can be found here.
The detection strategy was set according to article Grasping Primitive Enthusiasm - Approaching Primitive Obsession in Steps.
Here is a brief explaination:
Let's say the ratio of primitve type paramters to all parameters in a method is represented as Rm.
The ratio of all primitve parameters to all parameters in one class is RM.
We iterate through all methods in a class and mark thos whose Rm > RM.

Compliant Code Example

  class C1{
	  void m1(A a, B b, int c){}

	  void m2(D d, int e){}
  }

Noncompliant Code Example

  class C1{
	  void m1(double a, Integer b, int c){}

	  void m2(D d, int e){}
  }

Recommended Solution

Declare a new class to handle primitve data that might have further use.
