This rule detects data clumps between methods. When ever three or more of the same parameters appear in two or more methods as a group,
they will be detected as data clump. This rule uses parameter name for detection.

Noncompliant Code Example

	m1(int a, int b, int c){}
	
	m2(int a, int b, int d){}

Compliant Code Example

	void m1(int a, int b, int c, int d){}
	
	void m2(int a, int b, int c){}
	
	void m3(int b, int c, int d){}

m2 will be marked as data clump twice, once for group(a,b,c) and once for group(b,c,d)
m2 and m3 will be marked once.

Recommended Solution

Declare a new class with the clumped parameters as class variable. Pass the class object as parameter instead.
