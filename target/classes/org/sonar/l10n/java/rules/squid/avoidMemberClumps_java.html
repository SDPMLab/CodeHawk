This rule detects data clumps between classes. When ever 10 or more of the same variables appear in two or more classes as a group,
they will be detected as data clump. This rule uses variable name for detection.

Noncompliant Code Example

	class A{
		int a=0, b=0, c=0, d=0, e=0, f=0, g=0, h=0, i=0, j=0;
	}
	class B{
		int a=0, b=1, c=2, d=3, e=0, f=0, g=0, h=0, i=0, j=0;
	}
	class C{
		int a=0, b=0, c=0, d=0, e=0, f=0, g=0, h=0;
	}

class A and B will be marked as member clump for group(a,b,c,d,e,f,g,h,i,j), while class C will not.
the value of variables does not matter.

Recommended Solution

Declare a new class with the clumped variables as class variable and replace those variables with the class object.
